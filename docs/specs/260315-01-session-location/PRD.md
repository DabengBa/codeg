# 会话定位

## 🎯 1. 意图与核心体验 (Vibe & Intent)

**一句话描述**：在长会话中提供一个轻量、稳定、低打扰的轮次导航，帮助用户快速回到某次提问或该轮 AI 的最终答复。  
**核心用户路径**：用户打开长会话 → 点击消息区左上角“会话定位”入口 → 展开定位卡片 → 浏览每轮的“用户摘要 + AI 最终答复摘要” → 点击任一摘要 → 稳定跳转到对应位置，并通过短暂高亮快速确认命中目标后继续阅读。

---

## 📦 2. 范围与功能点 (Scope)

_让 AI 逐条实现并打勾_

### 核心功能 (MVP)

- [ ] **左上新增入口**：在消息区左上角增加一个与现有“计划”卡片视觉风格一致的可折叠悬浮入口，名称为“会话定位”。
- [ ] **按轮次生成目录**：目录项按“用户提问 + 该轮 AI 最终答复”组织，不逐条暴露 AI 的中间思考、工具调用和工具结果。
- [ ] **展示双摘要**：每个目录项最多展示两段摘要：`用户摘要` 与 `AI 最终答复摘要`，每段最多两行。
- [ ] **支持双目标跳转**：用户摘要和 AI 摘要分别可点击，点击后稳定跳转到各自对应的位置。
- [ ] **跳转后突出目标**：跳转完成后，目标内容需短暂高亮，帮助用户快速确认命中的具体位置。
- [ ] **支持待回复轮次**：当用户消息已经出现、但 AI 还没有稳定最终答复时，该轮仍应出现在定位目录中，至少可跳到用户消息。
- [ ] **支持降级文案**：当某轮没有可读文本时，使用明确的占位文案，例如“仅工具调用”“图片/附件”等。
- [ ] **默认低打扰**：入口默认折叠，不自动展开，不影响现有消息阅读、输入、滚动与“计划”卡片使用。
- [ ] **与计划浮层行为一致**：折叠/展开交互、悬浮卡片形态与现有“计划”浮层保持一致，不额外引入强制折叠或 click-away 逻辑。

### 明确不做 (Out of Scope)

- [ ] **不做中间过程目录**：不展示 AI 的 `thinking`、`tool_use`、`tool_result` 的逐步目录。
- [ ] **不做全文搜索/筛选**：本期不支持在定位卡片内搜索关键词、筛选角色、筛选时间。
- [ ] **不做滚动联动高亮**：本期不根据主消息区滚动位置自动高亮当前目录项。
- [ ] **不做书签/标记**：本期不支持用户手动收藏、置顶或编辑目录项。
- [ ] **不改后端存储模型**：不新增 Rust 端持久化字段，不修改会话数据库结构，不为“最终答复”增加后端标记位。

## ⚠️ 3. 边界与异常规则 (Edge Cases & Rules)

_这是 AI 最容易出错的地方，必须明确指示_

### 3.1 目录建模规则

- **一条目录项 = 一轮对话**：以“用户 turn”为起点，向后匹配最近的一条 AI assistant turn 作为同一轮的答复。
- **System 不进入目录**：系统消息不作为定位目录项，也不参与轮次配对。
- **孤立 assistant 不单独成项**：如果历史数据中出现没有前置用户消息的 assistant turn，本期不单独生成目录项，避免目录语义混乱。
- **连续用户消息要保留**：如果连续出现多条用户消息，前一条按“待回复轮次”保留，后一条继续作为新的目录项起点。
- **只有用户、没有 AI**：若用户已发出消息但后续没有 AI 稳定最终答复，该轮目录仍可显示用户摘要；AI 摘要显示为待回复状态文案或不显示。

### 3.2 “AI 最终答复”定义

- **最终答复不是 agent 内部标记**：本功能中的“AI 最终答复”是前端基于现有 turn 数据推导出的概念，不代表底层协议明确提供了 final 标识。
- **判定规则**：同一 assistant turn 中，从后往前找到的最后一个可读文本块，视为该轮 `AI 最终答复`。
- **不把 thinking/tool 当最终答复**：`thinking`、`tool_use`、`tool_result`、权限请求等都不视为“最终答复正文”。
- **无文本时降级**：如果某个 assistant turn 没有任何可读文本，则该轮 AI 摘要显示为“仅工具调用”或等价占位文案。
- **跳转目标兜底**：点击 AI 摘要时，优先跳到该轮 assistant turn 的最后一个正文文本；如果该轮没有正文文本，则跳到该 turn 的最后一条可渲染内容。
- **流式生成不视为稳定最终答复**：正在生成中的 assistant 回复不作为稳定最终答复参与目录摘要。

### 3.3 交互细节

- **样式参考现有计划卡片，但不是复制 hover 菜单**：视觉与结构参考现有“计划”悬浮卡片；交互上保持“可折叠浮层”一致，不引入新的 hover-only 行为。
- **不做 ClickAway 自动关闭**：为保持和现有“计划”卡片一致，展开后点击外部区域不会自动关闭，需通过折叠按钮收起。
- **摘要独立点击**：每个目录项中的“用户摘要”和“AI 摘要”都是独立的点击目标，不能只支持整项点击。
- **键盘可达**：所有可点击摘要必须可通过 `Tab` 聚焦，通过 `Enter` / `Space` 触发。
- **折叠状态只要求当前会话内可用**：本期不要求跨会话、跨页面刷新记忆“会话定位”的展开状态。
- **高亮反馈需自动消退**：跳转后的目标高亮应在短时间后自动消退，不需要用户手动关闭。

### 3.4 空态与异常状态

- **无可定位内容**：如果当前会话没有任何可构成目录的内容，则不显示“会话定位”入口，避免出现空壳浮层。
- **仅附件/图片消息**：如果用户消息没有文本，但存在附件、图片或资源引用，摘要使用明确占位文案，而不是空白。
- **超长摘要**：目录中的摘要必须截断显示，不能撑破卡片宽度或导致布局跳动。
- **窄宽度场景不单独定制**：本期不为窄宽度额外设计强制折叠策略；会话定位在此场景下仍沿用与“计划”浮层一致的行为模型。

### 3.5 性能红线

- **长会话可用**：在长会话下展开目录、点击跳转不应出现明显卡顿或白屏感。
- **不丢阅读上下文**：展开/收起定位卡片不应导致主消息区滚动位置异常丢失、闪动或重置。
- **不影响输入链路**：定位功能不能干扰现有消息输入、发送、生成中状态和计划卡片使用。

## ✅ 4. 验收标准 (DoD)

- [ ] 用户在长会话中能通过左上角“会话定位”入口展开目录，并看到按轮次组织的定位列表。
- [ ] 每个目录项至少能稳定展示用户摘要；存在 AI 文本时能展示该轮 AI 最终答复摘要。
- [ ] 点击用户摘要后，页面能跳转到对应用户消息；点击 AI 摘要后，页面能跳转到对应 assistant 消息。
- [ ] 跳转完成后，命中的目标内容会短暂高亮；若目标没有可高亮的正文 part，则至少高亮对应 turn 容器。
- [ ] 当用户消息已出现但 AI 仍未形成稳定最终答复时，该轮仍能在目录中找到并跳回用户消息。
- [ ] AI 的 `thinking`、`tool_use`、`tool_result` 不会被错误当作最终答复显示在目录摘要中。
- [ ] 对于仅工具调用、仅图片/附件、无文本答复、连续用户消息、孤立 assistant、无可定位内容等边界场景，行为符合本文定义。
- [ ] UI 样式与现有消息区设计语言一致，不引入明显违和的新浮层模式。
- [ ] 键盘可达、焦点可见，且不引入新的 hover-only 交互依赖。
- [ ] 打开、关闭、跳转过程中无明显卡顿；长会话中使用体验稳定。

## 5. UI 方案补充

### 5.1 视觉定位

- **位置**：位于消息区左上角，与现有右上角“计划”卡片形成左右呼应关系。
- **风格**：延续现有消息区悬浮卡片的轻量、半透明、低打扰气质，不额外创造新的视觉体系。
- **层级感**：定位卡片应被理解为“阅读辅助工具”，而不是新的主导航或侧边栏。

### 5.2 展开态

- 顶部标题栏包含：
  - 定位图标
  - 标题“会话定位”
  - 目录项数量
  - 折叠按钮
- 内容区为可滚动列表，每个目录项使用轻量分组，不使用厚重的强对比大色块。

### 5.3 目录项样式

- 每个目录项包含两行主体内容：
  - `用户` 摘要
  - `AI` 最终答复摘要
- 当 AI 尚无稳定最终答复时，允许只展示用户摘要，AI 行降级为待回复或占位提示。
- 摘要应短、稳、可扫读；过长内容必须截断，不应破坏整体布局。
- 占位文案应弱化处理，不与正常正文抢夺注意力。

### 5.4 交互反馈

- 折叠按钮、目录项行、摘要点击目标都必须具有清晰的 hover / focus 反馈。
- 跳转后不额外弹出干扰性反馈；以稳定、准确的定位和短暂高亮作为主要响应，不强依赖长距离平滑滚动动画。
- 与“计划”卡片同时存在时，两者不应破坏正文阅读与主要操作路径。

### 5.5 与现有架构/风格的一致性要求

- 不新增独立设计系统或新的浮层范式。
- 保持与现有消息区、计划卡片、按钮与文案语气的一致性。
- 避免做成厚重侧边栏、复杂目录树或高频动态面板。
