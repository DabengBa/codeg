# 会话定位 PR 演示文档

适用分支：`feature/session-location`

## 1. 功能摘要

本次分支为会话详情页新增了“会话定位”能力，用于在长会话中快速跳转到目标轮次，减少手动滚动查找成本。

核心表现：

- 在消息区顶部增加“会话定位”悬浮卡片，默认折叠为 `定位 {count}`。
- 展开后按“用户消息 + AI 回复”成对展示会话轮次摘要。
- 点击任一条目后，线程会自动滚动到对应位置，并在目标内容上显示一次外边框呼吸高亮。
- 与现有计划卡片共存时，窄宽度下自动纵向堆叠，避免覆盖。

## 2. 用户价值

- 长会话中快速回到某一轮输入或回复，无需手动滚动搜索。
- 通过摘要预览快速识别目标轮次，降低定位成本。
- 支持只有附件、只有工具输出、等待回复等非标准内容形态。
- 保持现有虚拟列表和消息渲染结构，不引入新的全局状态或额外页面入口。

## 4. 演示脚本

建议录制一段 60 到 90 秒的演示视频，顺序如下：

### 场景 A：折叠态与展开态

1. 打开一个包含多轮用户 / AI 对话的长会话。
2. 展示消息区左上方的折叠态按钮：`定位 {count}`。
3. 点击按钮展开“会话定位”卡片。
4. 展示卡片结构：
   - 顶部标题与轮次数量
   - 每组包含一条“用户”摘要和一条“AI”摘要

预期展示点：

- 会话定位默认不打断阅读，只占用顶部一块悬浮区域。
- 卡片内容可滚动，适配较长会话。

### 场景 B：跳转到用户消息

1. 在定位卡片中点击某条“用户”摘要。
2. 展示线程自动滚动到目标用户消息。
3. 展示目标内容块出现一次单外框呼吸高亮。

预期展示点：

- 跳转目标不是模糊的“附近区域”，而是对应用户内容块。
- 高亮只出现一个外层框，不会套出双层框。

### 场景 C：跳转到 AI 回复

1. 在定位卡片中点击某条“AI”摘要。
2. 展示线程跳转到 AI 最终回复内容。
3. 重点展示：高亮从目标回复内容开始，而不是把前面的 reasoning / 工具块一起包住。

预期展示点：

- AI 侧优先命中最后一个非空文本块。
- 若 AI 没有文本但有工具内容，也能跳到最后可渲染块。

### 场景 D：异常内容形态

可任选一个补充：

- 用户消息只有图片 / 附件，摘要显示为“图片/附件”
- AI 回复只有工具调用 / 工具结果，摘要显示为“仅工具调用”
- 用户消息已发出但 AI 尚未回复，定位卡片显示“等待回复”

预期展示点：

- 非文本消息不会让定位卡片变成空白项。
- 等待回复状态不会误导用户点击到不存在的 AI 内容。

### 场景 E：与计划卡片共存

1. 打开一个同时带有计划卡片和会话定位卡片的会话。
2. 先展示常规桌面宽度下，两张卡片分列在顶部左右两侧。
3. 再把窗口宽度缩窄到约 640px 以下，展示两张卡片自动纵向堆叠。

预期展示点：

- 两个 overlay 不会互相遮挡。
- 缩窄窗口后仍可点击和阅读。

## 6. 实际行为说明

### 6.1 摘要生成规则

- 用户摘要：取首个非空文本块。
- 用户无文本但有图片 / 资源：显示“图片/附件”。
- AI 摘要：取最后一个非空文本块。
- AI 无文本但有工具内容：显示“仅工具调用”。
- 无 AI 回复：显示“等待回复”。

### 6.2 配对规则

- 基于当前线程渲染数据生成目录项。
- `system` 消息不进入定位卡片。
- `streaming` 轮次不进入定位卡片。
- `optimistic` 阶段仅保留用户消息，用于表示“已提问但等待回复”的状态。

### 6.3 跳转与高亮规则

- 继续复用虚拟列表的程序化滚动能力，不引入 DOM 锚点导航。
- 跳转优先落到具体 `partIndex`；只有没有可用 `partIndex` 时才回退到整条 turn。
- 高亮为一次性外边框呼吸效果，不是持续选中态。

### 6.4 可访问性与响应式

- 行按钮的可访问名称保留可见 preview，不再只有通用动作文案。
- 会话定位与计划卡片共享顶部轨道，窄宽度下自动堆叠。
